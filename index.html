Concepts I've Learned (Git & GitHub)

  • Git is like a diary for your code; it keeps track of every change.
  • A repository is the notebook itself where your code lives.
  • git status shows which files are changed or ready to be saved.
  • git add stages files for commit, like selecting pages to file.
  • git commit permanently saves changes to history.
  • git log lets you see who changed what and when.
  • Branches allow experimentation without affecting main code.
  • Merging combines branches safely.
  • Real-world debugging: If an app stops connecting to the database, git log helps find recent configuration changes.

My Learning Summary

Git Basics

  • Git is like a time machine for your code; it takes snapshots called commits.
  • Staging lets you select changes to include in the next commit.
  • Branching allows you to try new features without affecting the main code.

Real-World Debugging Example

  • Used git blame to find who changed a config file when the app crashed.
  • GitHub is like a social network for code; pull requests let you discuss and merge changes safely.
  • Learning Git basics helps quickly debug and fix issues in DevOps environments.
